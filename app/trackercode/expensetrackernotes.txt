Function handler
The function handler takes deals with the input from the form submission later in the code.
The first two parts deal with validation:

event.preventDefault()
Stops the form’s normal submit behavior (page refresh), so your React handler can control the process.

if (!name.trim() || isNaN(Number(price))) { return }
This is the gatekeeper check:

!name.trim() means the name is empty or only spaces
isNaN(Number(price)) means price is not a valid number
If either is true, return exits early, so no bad expense gets created.

Then const newExpense only runs if validation passes; each item gets a new value, text is trimmed from empty spaces at either end, and price is stored as a number ready to be used in calculations.
The next part is where the expenses array has the previously made newExpense added to it, with name and price both set to empty spaces after, ready for the next submission. isMenuOpen is also set to false, triggering the input form to close.
